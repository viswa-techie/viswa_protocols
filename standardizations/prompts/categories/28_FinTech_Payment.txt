PAYMENT CARD INDUSTRY:
  PCI DSS v4.0 (2022)          — Payment Card Industry Data Security Standard
      12 requirements, 6 goals, 1,800+ sub-requirements
  PCI PIN (PCI PTS)            — PIN Transaction Security (ATM, POS terminal HSMs)
  PCI P2PE                     — Point-to-Point Encryption standard
  PCI 3DS                      — 3-D Secure authentication (Verified by Visa)
  PCI SSF                      — Software Security Framework (SPoC, SCRP)
  PCI CPP                      — Card Production and Provisioning standard
  PA-DSS (legacy)              — Superseded by PCI SSF for payment applications
  PCI Secure Software Standard — Application security for payment software

MESSAGING & INTEROPERABILITY:
  ISO 20022:2013               — Universal financial messaging standard
      MX messages replacing MT messages in SWIFT network
      Used by: SEPA, FedNow, CHAPS, SIC, BOJ-NET, RTP
  SWIFT MT messages            — Legacy financial messaging (being retired 2025)
  SWIFT MX messages            — ISO 20022-based (replacement for MT)
  SWIFT UETR                   — Unique End-to-End Transaction Reference (gpi)
  FIX Protocol                 — Financial Information eXchange (securities trading)
  FpML                         — Financial products Markup Language (derivatives)

EMV CHIP CARD:

  EMVCo Payment Tokenization     — Network token standard
  EMVCo Secure Remote Commerce  — E-commerce checkout standard

HISTORICAL CONTEXT TIMELINE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1950 — Diners Club card (first charge card)
1958 — BankAmericard (Visa predecessor) launched by Bank of America
1966 — Interbank (MasterCard predecessor) formed
1973 — SWIFT founded (Society for Worldwide Interbank Financial Telecommunication)
1975 — First ATM using magnetic stripe card (widespread)
1994 — SET (Secure Electronic Transaction) protocol — first attempt at secure e-commerce
         Failed due to complexity; never widely deployed
1999 — EMV migration begins in Europe (chip & PIN) after massive mag-stripe fraud
2000 — 3-D Secure 1.0 (Verified by Visa) launched — pop-up based, widely hated
2001 — First SWIFT messaging standard revision to ISO 15022
2002 — PCI founding: Visa/MC/Amex/Discover/JCB launch joint security program
2004 — PCI DSS v1.0 released (joint standard replacing individual card brand programs)
2008 — Heartland Payment Systems breach: 130M cards — largest at time
         → PCI DSS v1.2 emergency revision
2011 — EMV migration mandate USA announced (2015 liability shift)
2013 — Target breach: 40M cards via HVAC contractor pivot
         → PCI DSS v3.0 with vendor management requirements
2015 — US EMV liability shift; mag-stripe fraud moves to card-not-present (CNP)
2016 — Bangladesh Bank SWIFT heist: $81M stolen via SWIFT MT103 fraud
         → SWIFT CSP (Customer Security Programme) mandatory
2018 — PSD2 open banking APIs go live in EU (Strong Customer Authentication)
         GDPR enforcement begins — payment data as personal data
2019 — FIDO2 / WebAuthn W3C standard (passwordless banking)
2020 — ISO 20022 SWIFT migration program begins
         COVID accelerates contactless payment adoption
2022 — PCI DSS v4.0 (major revision: customized approach, MFA everywhere)
         EU MiCA crypto regulation published
2023 — SWIFT completes ISO 20022 MX migration (coexistence period 2022-2025)
         US FedNow instant payment system launches (ISO 20022 native)
         eIDAS 2.0 EU Digital Identity Wallet framework
2024 — PCI DSS v4.0 full compliance deadline (March 2025)
         EU DORA enforcement begins (January 2025)
         EU MiCA enforcement (December 2024)
2025 — EU AI Act impacts credit scoring and algorithmic trading
         CBDC pilots: digital Euro (ECB), digital Pound, e-CNY at scale

1. PCI DSS v4.0 New Requirements (effective March 2025):
   Req 6.4.3: JavaScript integrity on payment pages (SRI)
   Req 11.6.1: Change detection mechanism for payment pages
   Req 12.3.2: Targeted risk analysis for each customized approach control
   MFA: Now required for ALL access to cardholder data environment
   Customized Approach: Alternative to prescriptive requirements using compensating controls

2. EMV Chip Transaction Flow:
   Terminal capabilities exchange → Application selection (PSE/PPSE)
   → Card authentication (DDA/CDA) → Cardholder verification (PIN/signature)
   → Transaction authorization (online/offline) → Issuer scripts
   Cryptograms: ARQC (auth request), ARPC (auth response), TC (transaction certificate)

3. ISO 20022 Message Structure:
   Business Application Header (BAH) + Document payload
   Pain.001 (credit transfer initiation), Pacs.008 (FI-to-FI transfer),
   Camt.053 (bank-to-customer statement), Camt.056 (payment cancellation)

4. SWIFT CSP Mandatory Controls (2024 — CSCF v2024):
   8 Mandatory Controls: System hardening, authentication, anomaly detection,
   software updates, network segmentation, operator privilege restriction, etc.
   25 Advisory Controls: Best practice improvements

5. DORA 5 Pillars:
   1. ICT Risk Management Framework
   2. ICT-Related Incident Reporting (major incident → 4h initial / 72h final)
   3. Digital Operational Resilience Testing (TLPT — Threat-Led Penetration Testing)
   4. ICT Third-Party Risk Management (critical ICT providers → direct EU oversight)
   5. Information Sharing
