STANDARDS UNIVERSE:
SBOM FORMATS & STANDARDS:
  SPDX 3.0 (Linux Foundation, 2024) — Software Package Data Exchange
      Profiles: Software, Security, License, AI, Dataset, Build, Lint
      Formats: JSON-LD, YAML, XML, Tag-Value, RDF/Turtle
      SPDX 2.3 (2022): current widely deployed version
  CycloneDX 1.6 (OWASP, 2024)   — SBOM format for cybersecurity use cases
      Component types: libraries, frameworks, containers, devices, firmware
      Supports VEX (Vulnerability Exploitability eXchange)
      Also: SaaSBOM (services), OBOM (operations), HBOM (hardware)
  SWID Tags (ISO/IEC 19770-2:2015) — Software Identification Tags; ISO standard
  NTIA SBOM Minimum Elements (2021) — Supplier, component name, version, identifier,
                                       dependency relationships, author, timestamp
  VEX (Vulnerability Exploitability eXchange) — OpenVEX spec; CycloneDX VEX support

SBOM MANDATES:
  US Executive Order 14028 (2021) — SBOM for federal software procurement
  NTIA SBOM Guidance (2021-2023)  — Minimum elements, formats, sharing
  FDA Guidance 2023               — SBOM for medical device submissions
  EU Cyber Resilience Act (2024)  — SBOM for products with digital elements
  EU NIS2                         — Supply chain security (SBOM implied)
  DoD SBOM Guidance               — Defense contractor requirements

OPEN SOURCE LICENSES:
COPYLEFT:
  GPL v2 (1991)                  — GNU General Public License v2: share-alike for derived works
  GPL v3 (2007)                  — GPL v3: added patent protection, Tivoization prohibition
  LGPL v2.1 (1999)               — Lesser GPL: library linking without full copyleft
  LGPL v3 (2007)                 — Updated LGPL with GPLv3 terms
  AGPL v3 (2007)                 — Affero GPL: network use triggers copyleft (SaaS)
  MPL 2.0 (Mozilla, 2012)        — File-level copyleft; compatible with GPL
  EUPL v1.2 (2017)               — European Union Public License (multilingual)
  OSL 3.0                         — Open Software License (copyleft)

PERMISSIVE:
  MIT License                    — Extremely permissive; attribution only
  BSD 2-Clause / 3-Clause        — Similar to MIT; 3-clause: no endorsement
  BSD 4-Clause (original)        — Advertising clause (largely deprecated)
  Apache License 2.0 (2004)      — Permissive + explicit patent grant
  ISC License                    — OpenBSD preferred; similar to MIT
  Boost Software License 1.0     — C++ Boost library license
  Zlib License                    — Compression library; minimal restrictions
  Public Domain (CC0)            — Creative Commons Zero: no rights reserved

COPYLEFT ANALYSIS:
  Tivoization prohibition        — GPLv3 § 6: hardware installation information
  Dynamic vs. static linking     — GPL contamination debate
  Network clause (AGPL)         — SaaS GPL loophole closed
  License compatibility matrix  — GPL v2 vs. v3 vs. Apache 2.0 vs. MIT conflicts
  Copyleft strength spectrum     — CC0 → MIT → Apache → LGPL → MPL → GPL → AGPL

SUPPLY CHAIN STANDARDS:
  OpenChain ISO 5230:2020        — Open source license compliance program
      Minimum requirements for open source compliance
      Trusted open source provider certification
  OpenChain ISO 18974:2023       — Open source security assurance (SBOM)
  OWASP Dependency-Check         — Known vulnerable components scanner
  OWASP CycloneDX BOM Tools      — SBOM generation tools
  OWASP SAMM (Software Assurance Maturity Model) — Security practice maturity
