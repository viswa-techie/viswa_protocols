STANDARDS UNIVERSE:
NIST POST-QUANTUM CRYPTOGRAPHY (PQC) STANDARDS:
  NIST FIPS 203 (August 2024)   — ML-KEM (Module-Lattice Key Encapsulation Mechanism)
      Based on CRYSTALS-Kyber (IND-CCA2 secure KEM)
      Key sizes: ML-KEM-512, ML-KEM-768, ML-KEM-1024
      Security levels: ~AES-128, ~AES-192, ~AES-256 equivalent
      Replaces RSA/ECDH for key exchange
  NIST FIPS 204 (August 2024)   — ML-DSA (Module-Lattice Digital Signature Algorithm)
      Based on CRYSTALS-Dilithium (unforgeability under lattice hardness)
      Modes: ML-DSA-44, ML-DSA-65, ML-DSA-87
      Replaces RSA/ECDSA/EdDSA for digital signatures
  NIST FIPS 205 (August 2024)   — SLH-DSA (Stateless Hash-Based Digital Signature)
      Based on SPHINCS+ (hash-based; conservative security assumptions)
      Modes: fast (larger signatures) and small (slower)
      Backup signature if lattice broken
  NIST FIPS 206 (expected 2025) — FN-DSA (FALCON: Fast-Fourier lattice-based signature)
      Compact signature size; GPU-friendly verification
  NIST IR 8413 (2022)           — Status report on PQC candidates (Round 3)
  NIST SP 1800-38               — Migration to Post-Quantum Cryptography (draft)

LEGACY CRYPTOGRAPHY VULNERABILITIES TO QUANTUM:
  Shor's Algorithm (1994)       — Breaks RSA, ECC, DSA, DH in polynomial time
                                    → All current asymmetric cryptography is threatened
  Grover's Algorithm (1996)     — Quadratic speedup for symmetric key search
                                    → AES-128 → needs AES-256 for post-quantum security
  Harvest Now, Decrypt Later (HNDL) — Nation-states collecting encrypted data today
  NIST SP 800-131A Rev2         — Current algorithms; sunset dates for RSA/ECC
  Crypto agility                — Ability to swap algorithms without re-architecting
