SOFTWARE DEVELOPMENT SECURITY:
  NIST SSDF (SP 800-218) v1.1  — Secure Software Development Framework
      4 practice groups: Prepare the Org (PO), Protect Software (PS),
      Produce Well-Secured Software (PW), Respond to Vulnerabilities (RV)
  NIST SP 800-53 Rev5          — SA-11, SA-15 (software quality controls)
  OWASP SAMM v2.0              — Software Assurance Maturity Model
      5 business functions × 3 practices × 3 maturity levels
  BSIMM 14 (Building Security in Maturity Model) — Industry measurement (not prescriptive)
  SAFECode Fundamental Practices for Secure Software Development v3.0
  Microsoft SDL (Security Development Lifecycle) — Threat modeling, design review

SOFTWARE SUPPLY CHAIN & SBOM:
  NTIA Minimum Elements for SBOM (2021) — US CISA/NTIA guidelines
  SPDX 3.0 (ISO/IEC 5962:2021) — SBOM format + license data
  CycloneDX 1.6                — OWASP-backed SBOM + VEX format
  VEX (Vulnerability Exploitability eXchange) — CISA VEX guidance
  SLSA (Supply-chain Levels for Software Artifacts) v1.0
      SLSA Level 1: Provenance — build scripted
      SLSA Level 2: Hosted source, signed provenance
      SLSA Level 3: Hardened CI/CD, non-falsifiable provenance
  in-toto Framework            — Cryptographic attestation of supply chain steps
  Sigstore / cosign / rekor    — Signing and transparency log for artifacts
  GUAC (Graph for Understanding Artifact Composition) — Google supply chain graph

VULNERABILITY MANAGEMENT:
  CVE (Common Vulnerabilities and Exposures) — MITRE/NIST NVD
  CVSS v4.0 (Common Vulnerability Scoring System) — FIRST scoring standard
  EPSS (Exploit Prediction Scoring System) — Probability-based prioritization
  CWE (Common Weakness Enumeration) — Root cause taxonomy
  OVAL (Open Vulnerability and Assessment Language) — SCAP component
  SCAP 1.3 (Security Content Automation Protocol) — NIST automation standard
  CSAF 2.0 (Common Security Advisory Framework) — Machine-readable advisories

CONTAINER & CLOUD SUPPLY CHAIN:
  OCI (Open Container Initiative) — Image spec, runtime spec, distribution spec
  Sigstore / Cosign             — Container image signing
  SLSA for containers           — Build provenance for OCI images
  Chainguard / Wolfi            — Security-hardened container base images
  CIS Docker Benchmark          — Container configuration hardening
  Kubernetes RBAC / PSP (Policy) — Pod Security Admission standards

SECRET & CREDENTIAL MANAGEMENT:
  NIST SP 800-57                — Key management recommendation
  NIST SP 800-130               — Key management infrastructure
  HashiCorp Vault / OpenBao     — Secrets management (de facto standard)
  SOPS / age                    — Encrypted secret files in git

DEVSECOPS PIPELINE STANDARDS:
  OWASP ASVS (Application Security Verification Standard) v4.0
  OWASP Top 10 (2021)           — Most critical web application risks
  OWASP Top 10 CI/CD Security   — Pipeline-specific risks (2022)
  NIST SP 800-204 series        — Microservices security
  NIST SP 800-190               — Application container security
  CNCF Security Whitepaper v2   — Cloud-native supply chain security
  ISO/IEC 27034                 — Application security standard

COMPLIANCE & MANDATES:
  US Executive Order 14028 (May 2021) — "Improving the Nation's Cybersecurity"
      Mandates: SBOM for federal software, Zero Trust, MFA, EDR
  OMB M-22-18                   — Enhanced security requirements for software (EO 14028)
  EU Cyber Resilience Act (CRA 2024) — Product security requirements + SBOM for EU market
  EU NIS2 Directive 2022/2555   — Incident reporting, supply chain risk, penalties
  FedRAMP Rev5                  — Cloud security authorization (NIST SP 800-53 Rev5 based)
  CMMC 2.0 (DoD)               — Level 1/2/3 for defense contractor software supply chain
  PCI DSS v4.0 Req 6.3          — Software supply chain in payment context
  DORA (EU 2022/2554)           — ICT supply chain security for financial sector

THREAT MODELING STANDARDS:
  STRIDE (Microsoft)            — Spoofing, Tampering, Repudiation, Info Disclosure, DoS, EoP
  PASTA (Process for Attack Simulation and Threat Analysis)
  TARA (ISO/SAE 21434 for automotive)
  LINDDUN (Privacy threat model)
  MITRE ATT&CK for Enterprise   — TTP taxonomy used in threat modeling
  OWASP Threat Dragon           — Open-source threat modeling tool standard
  NIST SP 800-154               — Guide to Data-Centric System Threat Modeling

HISTORICAL CONTEXT
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

2001 — Gary McGraw publishes Software Security: Building Security In
2002 — Microsoft Security Push: internal SDL origin post-Windows XP vulnerabilities
2004 — BSIMM created (Gary McGraw / Cigital): first measurement framework
2008 — OWASP SAMM v1.0 published
2012 — Gartner DevSecOps term gains traction
2017 — Equifax breach (Apache Struts): supply chain vulnerability at scale
2019 — MITRE ATT&CK for Enterprise v5.0; supply chain TTP categorization
2020 — SolarWinds SUNBURST: nation-state software supply chain attack
         Build system (Orion build) compromised: 18,000 organizations affected
         → US government scrambles: NSA/CISA emergency guidance
2021 — US Executive Order 14028 (May): SBOM mandate, Zero Trust, MFA, EDR
         Log4Shell (December): Apache Log4j JNDI injection → supply chain SBOM urgency
         NTIA Minimum Elements for SBOM published
2022 — SLSA v1.0 launched (Google + OpenSSF)
         OWASP Top 10 CI/CD Security published
         OpenSSF Scorecard (automatic supply chain health scoring)
2023 — XZ Utils backdoor (CVE-2024-3094): malicious maintainer supply chain attack
         CISA SBOM guides published; CycloneDX 1.5/1.6
2024 — EU Cyber Resilience Act published: SBOM + product security for all products in EU
         NIST SSDF v1.1 officially references EO 14028
         Polyfill.io supply chain attack (millions of websites)
2025 — EU CRA enforcement period begins; SBOM becomes legal requirement in EU market
         AI-generated code supply chain risks: provenance of LLM-generated code
