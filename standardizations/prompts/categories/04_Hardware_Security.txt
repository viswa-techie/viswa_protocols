STANDARDS UNIVERSE:
CRYPTOGRAPHIC MODULE STANDARDS:
  FIPS 140-3 (2019)            — Security Requirements for Cryptographic Modules
      Level 1: Software; Level 2: Tamper-evident coatings; Level 3: Tamper-response;
      Level 4: Environmental attack protection
      Replaces FIPS 140-2 (testing sunset March 2026 for new submissions)
  ISO/IEC 19790:2012+Amd1:2015 — Security requirements for cryptographic modules (FIPS 140 international)
  ISO/IEC 24759:2017           — Test requirements for cryptographic modules
  CMVP (NIST)                  — Cryptographic Module Validation Program
  CAVP (NIST)                  — Cryptographic Algorithm Validation Program
  NIST SP 800-57 Parts 1-3     — Key Management Recommendations
  NIST SP 800-131A Rev2        — Transitions for cryptographic algorithms
  NIST SP 800-38 series        — Block cipher modes (CBC, CTR, CCM, GCM, XTS)
  FIPS 197 (AES)               — Advanced Encryption Standard
  FIPS 186-5 (DSS)             — Digital Signature Standard (RSA, ECDSA, EdDSA)
  FIPS 198-1 (HMAC)            — Keyed-Hash Message Authentication Code
  FIPS 202 (SHA-3)             — Keccak-based SHA-3 family
  FIPS 203 (ML-KEM/Kyber)      — Post-quantum key encapsulation
  FIPS 204 (ML-DSA/Dilithium)  — Post-quantum digital signature
  FIPS 205 (SLH-DSA/SPHINCS+)  — Post-quantum hash-based signature

PRODUCT SECURITY EVALUATION:
  Common Criteria (ISO/IEC 15408) — EAL 1 through EAL 7+
      EAL 4+: Bank card payment terminals, HSMs, smart cards
      EAL 6/7: Government/military encryption (rare)
  Protection Profiles (PPs):   — Domain-specific requirements
      PP for Mobile Device Fundamentals
      PP for Hardcopy Devices
      PP for IoT Device (NIAP)
      PP for General Purpose Operating Systems
  Security Target (ST)          — Vendor-provided security claims document
  NIAP (National Information Assurance Partnership) — US CC evaluation scheme

TRUSTED PLATFORM MODULE:
  TPM 2.0 Specification (TCG)  — Trusted Computing Group TPM Library Spec v1.83
      Part 1: Architecture; Part 2: Structures; Part 3: Commands; Part 4: Alg
      Algorithms: RSA 2048/3072, ECC P-256/P-384, SHA-256/384/512, AES-128/256
  ISO/IEC 11889:2015           — TPM 2.0 as ISO standard
  TCG PC Client Platform Firmware Profile — UEFI Secure Boot attestation
  TCG EFI Platform Spec         — PCR usage in UEFI boot
  TCG Storage Architecture Core — Self-encrypting drives (SED)
  TCGPC Client Reference Impl.  — Microsoft tpm2-tss, IBM TSS reference

HARDWARE SECURITY MODULES (HSM):
  FIPS 140-3 Level 3/4          — HSM certification (Thales Luna, Entrust, Utimaco)
  PCI HSM (PCI PTS)             — Payment HSM requirements
  EN 419221-5                   — Trustworthy Systems Supporting Server Signing
  Common Criteria for HSM       — PP for HSM used in trusted signing

SECURE BOOT & ROOT OF TRUST:
  UEFI Secure Boot (UEFI Forum) — Platform initialization trust chain
  Android Verified Boot (AVB 2.0) — Boot integrity for Android devices
  DICE (Device Identifier Composition Engine) — TCG spec for embedded RoT
  TCG SRTM (Static RTM)        — Measured boot using TPM PCRs
  TCG DRTM (Dynamic RTM)       — Intel TXT / AMD SEV late launch
  ARM TrustZone                 — Secure World / Normal World hardware separation
  ARM PSA Certified             — Platform Security Architecture certification
  NIST SP 800-193               — Platform Firmware Resilience (PFR) guidelines
  NIST SP 800-155               — BIOS Integrity Measurement Guidelines

SECURE ENCLAVES & TEE:
  GlobalPlatform TEE System Architecture — TEE client API, internal API
  GlobalPlatform Secure Element (SE) — NFC SE architecture
  Intel SGX (Software Guard Extensions) — Enclave attestation (DCAP)
  ARM TrustZone-A / TrustZone-M — AMBA TrustZone isolation
  RISC-V Physical Memory Protection — PMP/ePMP for secure zones

SIDE-CHANNEL ATTACK STANDARDS:
  ISO/IEC 17825:2016           — Testing for non-invasive attacks (TVLA)

  Common Criteria ALC_FLR      — Flaw remediation (for discovered side-channels)

JTAG SECURITY:
  IEEE 1149.1 — Standard boundary scan (disable in production)
  IEEE 1149.7 — Reduced-pin JTAG
  IEEE 1687 — iJTAG with access control
  JTAG protection: manufacturer disable, password protection, secure debug

HISTORICAL TIMELINE:
  1976 — Diffie-Hellman key exchange (asymmetric cryptography)
  1977 — RSA algorithm; DES FIPS 46 published
  1994 — Shor's algorithm (quantum threat to RSA/ECC)
  1997 — AES competition begins (NIST)
  1999 — Kocher: timing attack on RSA → side-channel field
  2001 — AES (Rijndael) selected as FIPS 197
  2004 — TPM 1.1b (TCG); Trusted Computing Platform Alliance → TCG
  2006 — TPM 1.2; Common Criteria EAL 4 becoming norm for payment
  2011 — FIPS 140-2 becomes dominant; NIST PQC awareness grows
  2014 — Heartbleed OpenSSL; hardware crypto modules gain urgency
  2016 — NIST PQC competition announced; ARM TrustZone-M (Cortex-M33)
  2019 — FIPS 140-3 published (ISO 19790-based)
  2022 — NIST PQC finalists: Kyber, Dilithium, FALCON, SPHINCS+
  2024 — FIPS 203/204/205 published (PQC standards official)
