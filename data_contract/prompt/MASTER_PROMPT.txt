CORE CONCEPTUAL ARCHITECTURE STACK
================================================================================

This is the single mental model that unifies all 15 volumes:

    ┌─────────────────────────────────────────────┐
    │            Application Objects              │  ← Business domain entities
    ├─────────────────────────────────────────────┤
    │            Data Modeling Layer              │  ← ER, UML, domain models
    ├─────────────────────────────────────────────┤
    │   Interface Definition Language (IDL)       │  ← .proto, AIDL, FIDL, Thrift
    ├─────────────────────────────────────────────┤
    │               Schema Layer                  │  ← JSON Schema, XSD, Avro, etc.
    ├─────────────────────────────────────────────┤
    │         Serialization / Encoding            │  ← JSON, CBOR, Protobuf wire fmt
    ├─────────────────────────────────────────────┤
    │           RPC / IPC Binding                 │  ← gRPC, Binder, D-Bus, COM
    ├─────────────────────────────────────────────┤
    │           Transport Protocol                │  ← HTTP/2, TCP, shared memory
    ├─────────────────────────────────────────────┤
    │            Network / Hardware               │  ← NIC, bus, physical layer
    ├─────────────────────────────────────────────┤
    │          Receiving Process / OS             │  ← kernel IPC, copy path
    ├─────────────────────────────────────────────┤
    │     Deserialization / Reconstruction        │  ← parse, validate, hydrate
    ├─────────────────────────────────────────────┤
    │      Application Object (Destination)       │  ← reconstructed entity
    └─────────────────────────────────────────────┘

CRITICAL DISTINCTIONS THAT MUST BE MAINTAINED THROUGHOUT:
  - IDL        ≠ Schema (IDL defines services + types; schema defines structure)
  - Schema     ≠ Serialization (schema describes; serialization encodes)
  - Serialization ≠ Wire Format (serialization is the act; wire format is bytes)
  - RPC        ≠ Transport (RPC is the call model; transport is the byte path)
  - IPC        ≠ Network protocol (IPC is process-boundary; network is machine-boundary)
  - Contract   ≠ Implementation (contract is the agreed interface; impl is one side)
  - Standard   ≠ Implementation (body defines spec; vendor implements it)
  - Zero-copy  ≠ Low-copy (true zero-copy never touches memory; low-copy minimizes)
